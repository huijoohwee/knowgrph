# Knowgrph Settings Catalog

## Schema Metadata & JSON-LD Context (Best Practice)
- Store shared provenance and GraphRAG configuration on the schema as top-level `schema.metadata`, not as per-type node/edge properties.
- Store JSON-LD serialization context on the schema as `schema.serialization.context`; JSON-LD schema export merges it into the document `@context`.
- The Schema UI editor exposes paired JSON editors for these fields via the global Metadata/Context row and the per-owner Template/Properties row: [SchemaUiEditor](file:///Users/huijoohwee/Documents/GitHub/knowgrph/canvas/src/features/schema/ui/SchemaUiEditor.tsx#L35-L465), [SchemaUiMetadataContextRow](file:///Users/huijoohwee/Documents/GitHub/knowgrph/canvas/src/features/schema/ui/SchemaUiEditorRows.tsx#L89-L133), and [SchemaUiTemplatePropsRow](file:///Users/huijoohwee/Documents/GitHub/knowgrph/canvas/src/features/schema/ui/SchemaUiEditorRows.tsx#L135-L170).
- JSON-LD schema IO: [schemaJsonLd](file:///Users/huijoohwee/Documents/GitHub/knowgrph/canvas/src/features/schema/schemaJsonLd.ts#L8-L158). Shared-schema linting (warns on metadata-like per-type properties): [lintSchemaMetadata](file:///Users/huijoohwee/Documents/GitHub/knowgrph/canvas/src/features/schema/validation.ts#L240-L288).

### UI Density: Icon presets and main panel shortcuts

- The primary surface for icon density and motion is the Settings tab under the **UI Density: Icons** area, which exposes `uiIconScale` (compact vs default) and `uiIconAnimationEnabled` as first-class settings backed by `registry-ui.ts` and the UI store.
- The main panel **Settings** header adds an inline “Icons” control strip with **Default** and **Compact** buttons that call `setUiIconScale('default' | 'compact')` via `useGraphStore`, plus a **Motion** toggle that flips `uiIconAnimationEnabled`. These controls write to the same fields as the Settings tab so users can preview icon density and motion changes in context without leaving the workflow.
- A **Compact preset** button in the same strip calls `setUiIconScale('compact')` and `setUiIconAnimationEnabled(true)` together, providing a one-click preset that tightens icon density and turns on launch/3D icon motion. This preset is intended as the default for teams who prefer the 3D Mode toolbar styling while keeping the underlying settings explicit in the catalog above.
- The **Graph Fields** tab reuses the same icon system: the legend icons (scope, origin, visibility, type) and the field list controls read `uiIconScale` and `uiIconStrokeWidth` via `getIconSizeClass` and store selectors, so Graph Fields badges and pills visually match the 3D Mode toolbar `IconButton` icons and stay in sync with the compact+motion presets.
- MainPanel and BottomPanel headers use an icon-only action strip sized by `uiIconScale` / `uiIconStrokeWidth`, ordered left-to-right as Search → Pin → Apply → Reset → Minimize/Restore → Close (skipping actions not available for the current panel/tab).
- BottomPanel tab selectors (Graph Data/Parser/Schema/Orchestrator/Renderer/History) are rendered as icon-only tabs with tooltips; they are sized by the same `uiIconScale` / `uiIconStrokeWidth` settings so they match header actions and the toolbar (`canvas/src/components/BottomPanel/BottomPanelHeader.tsx`, `canvas/src/features/panels/ui/TabHeader.tsx`).

### UI Density: Panel micro-label typography

- Panel micro-label helper text (for example, Advanced schema editor serialization errors, JSON editor validation messages, parser/schema/code error banners, and hover tooltip property descriptions) shares a dedicated text-size setting: `uiPanelMicroLabelTextSizeClass` in the shared UI slice, persisted via `LS_KEYS.panelMicroLabelTextSizeClass` and surfaced under the **UI Density: Panels** area in the Settings tab (`canvas/src/hooks/useGraphStore.ts`, `canvas/src/hooks/store/uiSlice.ts`, `canvas/src/features/settings/registry-ui.ts`, `canvas/src/features/panels/views/SettingsView.tsx`).
- When unset, `uiPanelMicroLabelTextSizeClass` falls back to the existing badge or helper sizing for each surface (for example, `uiIconBadgeChipTextSizeClass` for badge-adjacent tooltips or the previous `text-xs` size for legacy panel helpers), preserving existing typography while still allowing teams to tune micro-label density centrally.
- The **UI Density: Panels** presets in the Settings tab now include micro-label typography: the **Comfortable** preset sets `uiPanelKeyValueTextSizeClass='text-sm'`, `uiPanelMonospaceTextClass='font-mono text-xs'`, and `uiPanelMicroLabelTextSizeClass='text-xs'`, while the **Compact** preset tightens key/value rows and sets `uiPanelMicroLabelTextSizeClass='text-[9px]'` for denser helper text in panels, bottom-panel editors, and tooltips.

## Update Approach
- Maintain the catalog as the single source of truth for all configurable settings.
- Scan the repository end‑to‑end (frontend store/CSS/env, build tooling, backend env) and map each setting to a single S‑V‑O responsibility with precise module references and defaults.
- Render the catalog in the Panel’s Settings tab: make store‑backed keys writable; show build/backend entries as read‑only.

## Panel Settings Tab List View
- Show the top‑level list as: `Area | Key | Type | Value`.
- Expand a row to show: `Area | Modules | Classes/Objects | Functions/Methods | Responsibility | Dependencies / Imports | Notes`.

### Panel Settings Tab Directives
- Load details from `canvas/public/settings-flow.json`.
- Provide fallbacks for missing keys to guarantee `Area` and `Responsibility`.
- Provide header `Apply`/`Reset` icon buttons to batch actions; remove per-row Apply.
- Header `Search` input filters by `Area, Modules, Classes/Objects, Functions/Methods, Key, Responsibility, Dependencies/Imports, Notes`.
- Display non-writable entries for `CLICK_URL`, `PUBLIC_FALLBACK_JSON`, `KG_INPUT_PATH`, `KG_OUTPUT_DIR`, `max-lines`.
- Reuse the shared `KeyValueRow` two-column primitive for compact key/value rows in the Settings tab, keeping label/controls alignment consistent with Orchestrator traversal and GraphRAG workflow panels. Use `density="default"` for typical settings rows and opt into `density="compact"` only for high-density contexts such as traversal helpers and indexing parameters where many small controls must fit in a narrow panel.

## Notes

- Write responsibilities using S‑V‑O (e.g., `FeatureEngineer.create_domain_age_bins` generates domain age risk bins).
- Enforce globals as `window.document`, `window.navigator`, `window.location`, `window.localStorage`, `window.BroadcastChannel`.
- Clamp and persist numeric UI settings via `lsNum/lsSetNum` (`canvas/src/lib/persistence.ts:1–18`).
- Bind label LOD via `performance.lod.hideLabelsBelowScale`; adjust zoom visibility using `createZoom`.
- Expose schema options selectively in the Settings Panel; drive changes through schema setters.
- Inject frontend env via `process.env` / `import.meta.env`; manage values in `.env`.
- Document backend env (`KG_INPUT_PATH`, `KG_OUTPUT_DIR`) as reference only; do not expose in the Settings Panel.
