# Knowgrph Flight Sim Seed — Projection Companion Note

This is a **documentation note**, not a workspace seed. It has no `run_ready_demo` activation block and must never be applied as a seed. It records the conditional contract that would govern a future Agentic Canvas OS default-storage projection after promotion; no projection file currently exists.

## Authority and direction

- **Single authored source (edit here):** `knowgrph/docs/workspace-seeds/knowgrph-game-flight-sim-demo.md`. Its planned `run_ready_demo.id` (`flight-sim`), source identity, native flight demo, asset pipeline, camera source, Motion Control boundary, and Flight Sim projection are edited here first, per `knowgrph/docs/workspace-seeds/README.md` (Workspace Seed Authority). The draft carries `planned_run_ready_demo`, so it is not an activation authority before the required gates pass.
- **Future release-pinned projection (not generated):** `agentic-canvas-os/docs/workspace-seeds/knowgrph-game-flight-sim-demo.md`. If the draft is promoted after its gates pass, the protected release controller may generate a byte-identical default-storage projection. It will not be an independent authoring surface.
- **Direction of truth:** authored source → protected docs refresh → byte-identical projection. Never the reverse; never a parallel edit on the projection.

## Projection contract

- If generated after promotion, the projection must be **byte-identical** to the authored source and may be refreshed **only from this source**.
- Cross-repository validation must reject any byte drift after such a projection exists.
- Publish repositories must **not** contain an editable `docs/workspace-seeds` copy. Their runtime assets and public routes are generated by the protected release controller from the verified Knowgrph source.
- Stale, renamed, fallback, legacy, conditional, or conflicting projection variants are **forbidden** rather than aliased or hidden.
- **Native, in-repo, zero external dependency target:** the authored seed's `planned_run_ready_demo.external_dependencies` is empty, no future runtime remote asset/provider/model call is permitted, and no external project source may be copied or depended upon. A future projection must preserve these requirements.

## Current status

- **`projection_status`: not-generated.** The Flight Sim module is `draft` with no runtime-readiness proof yet, so no projection has been (or should be) generated. Generating the byte-identical projection is deferred to the protected release controller after:
  1. the Flight Sim runtime passes repository-owned runtime-readiness and browser-smoke commands, which are not implemented or registered yet, and
  2. an operator explicitly authorizes a protected release from an integrated SHA.
- Until then this note is the only companion artifact; the projection file is intentionally absent.

## Asset-pipeline and invocation guarantees carried into the projection

Any future projection must preserve the authored seed's planned guarantees verbatim (see the source seed and `docs/documents/knowgrph-game-flight-sim-prd-tad.md`):

- **Planned assets:** img2threejs small, diffable TypeScript + JSON scene specs must be the primary committed representation; a TRELLIS.2 opaque binary GLB may be a committed local fallback only. No runtime image-to-3D model, network fetch, or Cloudflare resource may be invoked to obtain an asset.
- **Planned invocation:** strict native `/flight.sim @canvas #flight` and browser-local WebMCP `knowgrph.inspect_local_flight_sim` / `knowgrph.control_local_flight_sim`. These are not registered while the source remains draft; their implementation must add no stdio, HTTP, gateway, or deployment surface.
- **Planned runtime:** browser-based, mobile-first, local-first, offline-first, deterministic, Decisions-only WorkspaceFs persistence; no sign-in, network, or Cloudflare service required for core play.

## Release boundary

No Pages build upload, Worker deployment, D1/R2/KV/DO mutation, production route change, or release claim belongs to this scope. Any future projection may be created or refreshed only by the protected release controller from a verified, operator-authorized source revision.
