# Knowgrph Flight Sim Seed — Projection Companion Note

This is a **documentation note**, not a workspace seed. It has no `run_ready_demo` activation block and must never be applied as a seed. It records the projection contract that governs how the authored Flight Sim seed reaches the publish repository, so the two files stay in a single-source, byte-identical relationship.

## Authority and direction

- **Single authored source (edit here):** `knowgrph/docs/workspace-seeds/knowgrph-game-flight-sim-demo.md`. Its planned `run_ready_demo.id` (`flight-sim`), source identity, native flight demo, asset pipeline, camera source, Motion Control boundary, and Flight Sim projection are edited here first, per `knowgrph/docs/workspace-seeds/README.md` (Workspace Seed Authority). The draft carries `planned_run_ready_demo`, so it is not an activation authority before the required gates pass.
- **Release-pinned projection (generated, do not hand-edit):** `agentic-canvas-os/docs/workspace-seeds/knowgrph-game-flight-sim-demo.md`. It is a byte-identical, release-pinned default-storage projection of the authored source. It is **not an independent authoring surface**.
- **Direction of truth:** authored source → protected docs refresh → byte-identical projection. Never the reverse; never a parallel edit on the projection.

## Projection contract

- The projection is **byte-identical** to the authored source. A protected docs update may refresh it **only from this source**.
- Cross-repository validation must **reject any byte drift** between the authored source and the projection.
- Publish repositories must **not** contain an editable `docs/workspace-seeds` copy. Their runtime assets and public routes are generated by the protected release controller from the verified Knowgrph source.
- Stale, renamed, fallback, legacy, conditional, or conflicting projection variants are **forbidden** rather than aliased or hidden.
- **Native, in-repo, zero external dependency:** the authored seed's `planned_run_ready_demo.external_dependencies` is empty, no runtime remote asset/provider/model call is permitted, and no external project source may be copied or depended upon. The projection inherits these guarantees unchanged.

## Current status

- **`projection_status`: not-generated.** The Flight Sim module is `draft` with no runtime-readiness proof yet, so no projection has been (or should be) generated. Generating the byte-identical projection is deferred to the protected release controller after:
  1. the Flight Sim runtime passes its local runtime-readiness gate (`npm run game-flight-sim:runtime-ready`, `npm run game-flight-sim:browser-smoke`), and
  2. an operator explicitly authorizes a protected release from an integrated SHA.
- Until then this note is the only companion artifact; the projection file is intentionally absent.

## Asset-pipeline and invocation guarantees carried into the projection

The projection preserves the authored seed's guarantees verbatim (see the source seed and `docs/documents/knowgrph-game-flight-sim-prd-tad.md`):

- **Assets:** img2threejs small, diffable TypeScript + JSON scene spec is the primary, committed representation; a TRELLIS.2 opaque binary GLB is a committed local fallback only. No runtime image-to-3D model, network fetch, or Cloudflare resource is invoked to obtain any asset.
- **Invocation:** strict native `/flight.sim @canvas #flight` and browser-local WebMCP `knowgrph.inspect_local_flight_sim` / `knowgrph.control_local_flight_sim`; no new stdio, HTTP, gateway, or deployment surface; the private Agentic ECS stdio lane remains exactly three tools.
- **Runtime:** browser-based, mobile-first, local-first, offline-first, deterministic, Decisions-only WorkspaceFs persistence; no sign-in, network, or Cloudflare service required for core play.

## Release boundary

No Pages build upload, Worker deployment, D1/R2/KV/DO mutation, production route change, or release claim belongs to this scope. The projection is created and refreshed exclusively by the protected release controller from a verified, operator-authorized source revision.
