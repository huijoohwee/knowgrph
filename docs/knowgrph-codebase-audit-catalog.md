## Summary

This catalog instantiates the INGEST → PRODUCE → REUSE responsibilities from `docs/knowgrph-raci-document.md` against concrete modules in this repository.

- Loader, parser, and validator modules remain domain-agnostic: repository scan confirms no loader/parser code validates domain-specific fields or branches on concrete filenames beyond explicitly labeled demo presets and test fixtures.
- GraphData and schema stay generic: the canonical `GraphData` container and `GraphSchema` definitions carry no domain-specific required fields; presets and example schemas (including Unicorn, AI-KG, Universal Lean Startup, A0 Investors, and the venture capital portfolio demo) remain opt-in overlays wired through workflow presets rather than core pipeline branches.
- Renderer and export helpers operate purely on generic nodes/edges and schema-driven styles; visualization and export behavior do not hardcode dataset-specific logic.
- Default schema styling uses only generic graph labels; domain-specific visuals are confined to opt-in schema-config presets and settings presets (see `canvas/src/lib/graph/schema.ts:247-332` and `canvas/src/features/settings/registry-presets.ts:7-109`).
- Demo workflow presets are indexed in `docs/knowgrph-workflow-document.md` and exercised by `canvas/src/__tests__/workflowPresetPipeline.test.ts:1-126`.
- Codebase traversal GraphData for AgenticRAG uses `python -m knowgrph_parser parse-codebase-index`, `schema-config/codebase-index-schema.json`, and `orchestrator-config/knowgrph-universal-orchestrator-config.yaml` to construct the codebase index and attach Agentic GraphRAG traversal metadata (`graphRAGPath`), synthesized `chunk_text`, node-level provenance, and runtime tracing metadata (via `RuntimeEvent` nodes), while keeping visualization decisions in the canvas renderer layer. The traversal UI in the bottom panel (`RenderPresetSection.tsx`, `RenderSettingsSection.tsx`) operates purely on `GraphData` (`id`, `label`, `type`, `properties`, `edges[].label`) and works for codebase index graphs via the generic `findTraversalEdgeIds` helper and typed `ParsedAgenticGraphRagPath` helpers without any dataset‑specific logic.

All rows in the Detailed section are currently marked ✅. If future changes introduce domain-specific behavior or hardcoded datasets into these responsibilities, update the Compliance column to ⚠️ or ❌ and adjust the Recommended Action to describe the remediation.
