## Goal

Publish Knowgrph Canvas to Cloudflare Pages path `airvio.co/knowgrph` by building with `base=/knowgrph/` and syncing build output into:

- `/GitHub/huijoohwee/content/knowgrph`

## One-command publish

From `/GitHub/knowgrph`:

```bash
chmod +x ./scripts/publish-to-huijoohwee.sh
./scripts/publish-to-huijoohwee.sh
```

Then commit + push in `/GitHub/huijoohwee`:

```bash
cd ../huijoohwee
git add content/knowgrph _redirects _headers
git commit -m "Publish knowgrph"
git push
```

## Fixes applied (so publish is clean)

### 1) TS2448: “used before its declaration”

If you see:

> `MarkdownBlockContainer.tsx ... TS2448: buildReplacementLinesFromDraft used before its declaration`

Fix is to ensure `buildReplacementLinesFromDraft` is defined before it appears in any hook dependency arrays.

### 2) esbuild css minify warning: `${selectionFlashColor}` in Tailwind class

If you see:

> `Expected identifier but found "$" [css-syntax-error] ... ${selectionFlashColor}`

Cause: Tailwind scanned a literal class string like `ring-[color:${selectionFlashColor}]` (template string), producing invalid CSS.

Fix: use a CSS variable in the class, and set the variable via inline style:

- `ring-[color:var(--kg-selection-flash-color)]`
- `style={{ '--kg-selection-flash-color': selectionFlashColor }}`

## Notes

- `KG_SKIP_DOCS_UPDATE=1` is used to avoid Python deps during publishing.
- Chunking warnings are OK for deploy; they are size warnings, not correctness failures.
