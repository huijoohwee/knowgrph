## Tokenization, Stopwords, and Stats Consistency

- The stats tab reuses the same neutral tokenization configuration as the semantic layer derivation:
  - `getStatsTokenizationConfig(schema: GraphSchema)` reads `schema.layers.semantic.textKeys`, `minTokenLength`, `maxTokensPerNode`, and `stopwords` from the active schema and clamps values to safe ranges. This helper lives in the feature-scoped stats module (`canvas/src/components/BottomPanel/BottomPanelStatsUtils.ts`) so both the Stats tab and any future semantic-aware panels can share one implementation.
  - `buildTokenFrequenciesForNodes` concatenates node labels and configured text fields, lowercases, tokenizes, and filters out short tokens and stopwords before counting frequencies. It is also defined in `BottomPanelStatsUtils.ts` alongside `tokenizeForStats`, `topTokenList`, `getEdgeWeightForStats`, `getEdgeCooccurrenceForStats`, and `formatNumber` so all Stats tab distributions reuse the same neutral pipeline.
- In-session stopword overrides are tracked via `statsExcludeTokens`, `statsIncludeTokens`, and `statsFilterMode`:
  - Token chips in the **Word frequencies by polygon**, **Word frequencies by node**, and **Communities (Louvain)** sections call `toggleStatsStopword`. In **Exclude words** mode this toggles tokens in `statsExcludeTokens` (case-insensitive). In **Include words** mode this toggles tokens in `statsIncludeTokens`.
- Token pickers in the polygon/community sections use a tickbox list (with filter/sort controls and a **Select all** checkbox) and share a global include/exclude mode (`statsFilterMode`).
  - In **Exclude words** mode, checked means “keep this token”; unchecked means “exclude this token” (adds it to `statsExcludeTokens`).
  - In **Include words** mode, checked means “include this token” (adds it to `statsIncludeTokens`), which also removes tokens from the effective stopword set even if they appear in `schema.layers.semantic.stopwords`.
  - The effective stopword set is `(schema.layers.semantic.stopwords ∪ statsExcludeTokens) \\ statsIncludeTokens`, so UI-driven exclusions and inclusions stay consistent with AgenticRAG JSON-LD directives while still allowing curators to temporarily surface schema-level stopwords such as “with”, “for”, or “and”.
- Because communities reuse the same tokenization pipeline, changes to the effective stopword configuration immediately propagate to community names, descriptions, and top-token chips without introducing any domain-specific presets.

## Selection, Highlighting, and Renderer Integration

- All stats sections derive their data from either the full dataset or a selection subgraph:
  - `statsScope` (`'auto' | 'dataset' | 'selection'`) and `statsLod` (`'auto' | 'low' | 'medium' | 'high'`) control which graph slice is used and how many items are considered for each list.
  - `normalizeSelectionIds` and `buildSelectionSubgraphForAnchorIds` construct a selection-aware `GraphData` before any stats computation, keeping logic selection-driven but schema-agnostic.
- Hover and click interactions are wired to the shared selection model:
  - Bar charts call `selectNodeIds` or `selectEdgeIds` to update the global selection, which is consumed by the Canvas, Graph Data Table, and other panels.
  - When a node/edge selection is made from the Canvas (or other selection surfaces), the matching distribution bars are highlighted and the chart horizontally scrolls to keep the selected/pinned bar visible.
  - Selection snapshots are captured and restored around pin/unpin interactions so temporary states do not overwrite pinned selections.
  - Clicking a polygon/community/edge bar pins selection for that item while keeping the main distributions computed from the pre-pin stats scope; when more detail is available (top tokens), a sub-bar chart renders below the main distribution without resizing the main chart.
- Small slider icon buttons in the **Polygon**, **Top semantic edges**, and **Community sizes** headers dispatch `emitRendererPanelOpen`, which opens the FloatingPanel Renderer:
  - This keeps semantic configuration (`schema.layers.mode`, `schema.layers.semantic.*`) and visualization settings aligned between the Stats tab, Render panel, and AgenticRAG JSON-LD schemas.

## Example flow: Polygon word-frequency chart

- The user opens the Stats tab and leaves `statsScope` on **Auto**. If a selection subgraph exists, `BottomPanelStatsTab` uses it as `effectiveGraph`; otherwise it uses the full dataset returned by `deriveGraphDataForLayers`.
- `getStatsTokenizationConfig(schema)` reads `schema.layers.semantic.textKeys`, `minTokenLength`, `maxTokensPerNode`, and `stopwords`. The UI merges `statsExcludeTokens` and `statsIncludeTokens` into this base configuration via `statsFilterMode` to derive an effective stopword set used by all token-frequency computations.
- `buildNodeGroupsFromSchema` scans `GraphData` and groups nodes into polygon memberships. For each group, `buildTokenFrequenciesForNodes` runs the tokenization pipeline over member nodes using the effective tokenization config, producing `tokensByPolygon` rows with `totalTokens`, `nodeCount`, `nodeIds`, and `topTokens`.
- The polygon bar chart passes one entry per `tokensByPolygon` row into `MiniBarChart`, so the chart always shows a bar for every polygon. Hovering a bar shows the SVG tooltip text (via `<title>`). Clicking a bar pins selection for that polygon while keeping the main distribution unchanged; when `topTokens` exist, a sub-bar chart renders below the main distribution without resizing the main chart.
- Clicking a polygon bar pins selection for that polygon. Clicking the same bar again (or clicking the background of the semantic edges chart when an edge is pinned) clears the pinned id and restores the previous snapshot via `restoreSelectionSnapshot`, keeping stats, canvas, and table selection in sync.
- Token chips under each polygon and the shared dropdown include/exclude controls mutate `statsExcludeTokens` and `statsIncludeTokens`. This recomputes the effective stopword set and immediately updates token counts and labels in the polygon, node, and community sections while leaving the underlying `GraphData` unchanged and respecting the AgenticRAG JSON-LD semantic configuration.
