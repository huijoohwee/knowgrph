# Summary: Complete Knowledge Graph Pipeline

This project implements a **Complete Knowledge Graph Pipeline** for transforming structured and semi‑structured content into interactive, analyzable knowledge graphs.

- **Universal CSV Schema Pattern (A0)** is the superset schema that unifies A1–A999 patterns.  
- A0 allows flexible extensions and supports mapping Markdown content into RDF triples.  
- Example source: [Chapter Summaries](https://github.com/chiphuyen/aie-book/blob/main/chapter-summaries.md).  
- Flow: Markdown → CSV (A0 schema) → JSON‑LD → RDF knowledge graph.

---

## Repository Layout & End-to-End Workflow

- `canvas/`: React + TypeScript canvas app; 2D (SVG) and 3D (Three.js/React Three Fiber) knowledge‑graph viewer.
- `parser-script/`: Python utilities that read CSV/JSON/JSON‑LD from `test-data/` and emit normalized `GraphData` or `{nodes,edges}` JSON into `test-data/` or `data/outputs/`.
- `test-data/`: Sample datasets and fixtures, including `unicorn-investors-top-3-test.json`, `unicorn-investors-top-3-test.csv`, and `ai-kg-viz.json`.
- `schema-config/`: Schema and render configuration JSON files used by the Schema/Settings panels (for example `unicorn-investors-test-3d-viz-schema.json`, `ai-kg-viz-schema.json`).
- `data/outputs/`: Generated graph artifacts produced by parser scripts and regression tests; useful for offline inspection and regression baselines.
- `catalog/`: Markdown catalogs for parsers, schema, visualization, workflows, user journeys, settings, tests, and utilities; these mirror the codebase’s end-to-end flow so contributors can navigate from high-level responsibilities to concrete modules.

- End-to-end flow (in this repo):
  - Parser scripts normalize raw CSV/JSON/JSON‑LD from `test-data/` into graph‑shaped JSON stored under `test-data/` or `data/outputs/` (`parser-script/`, see `catalog/knowgrph-workflow-catalog.md#user-journey`).
  - Toolbar loader reads CSV/JSON/JSON‑LD from `test-data/` in the canvas app (`canvas/`) and commits `GraphData` into the global store using built‑in parsers.
  - Schema and Settings panels import schema‑config JSON from `schema-config/` to apply node/edge styles, 2D layout forces, and 3D `three` settings, then export updated schema JSON/JSON‑LD/CSV back into `schema-config/`.
  - Canvas renders `(parser, dataset, schema)` combinations in 2D (`GraphCanvas`) or 3D (`ThreeGraph`), runs traversal presets (AI‑KG) via the Render panel, and lets users tune layout and visual presets.
  - Export graph data (JSON‑LD/JSON/combined CSV) and schema configs (JSON/JSON‑LD/CSV) back into `test-data/` and `schema-config/` so workflows stay reproducible across sessions and pipelines.

## Performance & Cleanup Guidelines

- Goals
  - Avoid conflicting/hardcoded/stale/duplicate code; prevent re-render storms, infinite loops, and leaks
  - Enhance cache/memory through memoization and selectors
- Caching
  - Search and parser caches key by stable inputs; invalidate on graph/history change (`canvas/src/features/search/search.ts`, `canvas/src/features/toolbar/utils.ts`)
  - Use TTL or versioned keys: `graphId|historyIndex|query|limit`
- Memoization
  - Derivations in tables use `useMemo`/`useCallback` and stable deps to avoid recompute on each keystroke (`canvas/src/components/BottomPanel/*.tsx`)
  - Large JSON stringify scheduled on idle where available; debounce inputs (`canvas/src/features/code-editor`)
- Zustand selectors
  - Read specific slices with selectors to minimize re-renders (`canvas/src/hooks/useGraphStore.ts` slices)
  - Prefer action creators over inline anonymous updaters to keep function identities stable
- Workers
  - Heavy parsing is offloaded to web workers (`canvas/src/workers/*.worker.ts`); ensure imports use `@` alias
- Effects hygiene
  - Always return cleanup functions from `useEffect`; remove listeners and cancel timers
  - Guard async setState with mounted flags in long-running operations
- SVG/Canvas lifecycle
  - Keep a single SVG node sized by container (`useContainerDims`); avoid re-creating DOM trees on every update
- Build hygiene
  - Keep `vite.config.ts` alias in sync with `tsconfig.json` `paths`
  - Document environment differences (dev vs preview) directly in README (this section)

### Schema Derive Cache & Helpers
- Helpers: `canvas/src/features/schema-editor/utils.ts`
  - `normalizeQuery`, `filterStringsByQuery`, `computeFilteredLists(data, schema, query)`
- Caching: `canvas/src/features/schema/derive.ts`
  - LRU caching for `uniqueNodeTypes`, `uniqueEdgeLabels`
  - Dynamic capacity with eviction; parameterized via Settings key `schemaDeriveCacheCapacity` (default 16)
  - Store field and setter: `schemaDeriveCacheCapacity`, `setSchemaDeriveCacheCapacity` in `canvas/src/hooks/store/uiSlice.ts`
  - Settings registry entry: `canvas/src/features/settings/registry.ts` (`schemaDeriveCacheCapacity`)
  - Persisted under `kg:perf:schemaDeriveCacheCapacity`

### Module organization
- Extract utilities into feature‑scoped modules to keep files under 600 lines and enforce single-responsibility by structuring each class or major component inside a feature‑scoped module.
- Use each primitive type (components, functions, patterns, abstractions) to structure reusable building blocks and enable higher‑level composition.

### Code hygiene
- Remove conflicting code, hardcoded values, stale code, corrupted code, duplicate code, legacy keys, redundant computation, unnecessary re‑render logic, infinite loops, memory leaks, unreferenced code, and unused props as you touch related modules.

## Python Parser AST Integration

- Overview
  - Python parsing supports two paths: a fast line-based parser and a higher-accuracy AST path via `web-tree-sitter` + `tree-sitter-python` in a Web Worker.
- Enable
  - Opt-in via `localStorage.setItem('parser.useTreeSitter', '1')` and reload.
  - With the flag on, `Apply Parser` for Python uses the AST worker by default in Loader and Parser tab.
- Modules
  - Spec: `canvas/src/features/parsers/python/index.ts` (`match`, `parse`, `parseAsync`)
  - Worker client: `canvas/src/features/parsers/python/tsWorkerClient.ts`
  - Worker: `canvas/src/features/parsers/python/treeSitterWorker.ts`
  - Flag: `canvas/src/features/parsers/config.ts` (`isTreeSitterEnabled`)
  - Registry: `canvas/src/features/parsers/registry.ts` (`applyParserAsync`)
  - Loader/Panel: `canvas/src/features/parsers/loader.ts`, `canvas/src/features/panels/views/ParserView.tsx`
