## Summary

This catalog maps the RACI roles from `catalog/knowgrph-raci-catalog.md` onto concrete code in this repository.

- Loader responsibilities (file/API/DB fetch, JSON syntax validation, passing JSON to parsers) are implemented in generic utilities without domain-specific field checks or filename-based branches.
- Parser and validator responsibilities focus strictly on structure: they normalize inputs into `GraphData` and enforce ID/reference/type rules, while treating properties and labels as opaque, schema-configured data.
- Schema, rendering, and styling responsibilities are driven by `GraphSchema` and presets; visual behavior is property-based rather than domain-specific.
- Default schema styling now uses only generic graph labels; domain-specific visuals live in opt-in schema-config presets and settings presets (see `canvas/src/lib/graph/schema.ts:247-332` and `canvas/src/features/settings/registry-presets.ts:7-109`). Demo datasets (Unicorn, AI-KG, Universal Lean Startup, A0 Investors, venture capital portfolio) are wired exclusively through workflow presets and the examples catalog, so RACI roles remain unchanged when new demos are added (`canvas/src/features/parsers/examplesCatalog.ts:18-49`, `canvas/src/features/parsers/useParserWorkflowState.ts:47-101`, `canvas/src/__tests__/workflowPresetPipeline.test.ts:54-126`).
- Workflow presets are indexed in `catalog/knowgrph-workflow-catalog.md` and mapped to onboarding click paths in `catalog/knowgrph-userjrny-catalog.md`.

As of 2025‑12‑16, a repository scan confirms that all rows marked ✅ remain compliant with the Loader/Parser/Validator/GraphData/Exporter/Renderer/Schema/Data constraints in the RACI catalog. If future changes introduce domain logic into these roles, update the Compliance and Recommended Action columns accordingly.
