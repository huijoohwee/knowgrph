# Knowgrph Workflow Catalog

See also: [Parser Catalog](knowgrph-parsers-catalog.md), [Visualization Catalog](knowgrph-viz-catalog.md), [User Journey Catalog](knowgrph-userjrny-catalog.md).

This catalog describes the end-to-end flow across `parser-script/`, `test-data/`, `schema-config/`, and the `canvas/` app so contributors can connect offline parser scripts, in-browser parsers, schemas, and visualization presets into reproducible workflows.

### Type‑Safe Workflow Paths
- Preset pipeline returns `(parserSpecId, datasetFileName, schemaFileName)` using branded types: `ParserId`, `DatasetPath`, `SchemaConfigPath`.
- Graph exporters accept typed suggestions: `saveGraphFile(data, DatasetPath)`, `exportGraphAsJSON(data, DatasetPath)`, `exportGraphAsCombinedCSV(data, DatasetPath)`.
- Schema exporters accept `SchemaConfigPath` suggestions: `exportSchemaAsJSON(schema, SchemaConfigPath)`, `exportSchemaAsJsonLd(schema, SchemaConfigPath)`, `exportSchemaAsCsv(schema, SchemaConfigPath)`.
- Integration test validates `unicorn-top3-3d` and `ai-kg-viz` pipelines: `canvas/src/__tests__/workflowPresetPipeline.test.ts`.

## User Journey

### Interactive Product Tour & Onboarding Flow

### Step 1 – Parse Data
- Run parser scripts to convert raw CSV/JSON into graph format (`{nodes, edges}`).
- Input/output flags let users import datasets and export normalized graph JSON.

### Step 2 – Import / Export Datasets
- Load CSV/JSON/JSON‑LD files via the **Load Data** button.
- Auto‑detects format, commits graph to store.
- Export current graph as JSON, JSON‑LD, or CSV with enforced naming rules.

### Step 3 – Apply Schema Config
- Open **Schema tab** to apply presets from `/schema-config`.
- Manage presets (Apply, Clear, Reset).
- Tune rules for nodes, edges, and layout.
- Customize UI with Schema UI Editor (styles, validation, layout forces).
- Export updated schema as JSON, JSON‑LD, or CSV.

### Step 4 – Visualize Graphs
- Combine parser + dataset + schema for reproducible workflows.
- Switch between **2D force‑directed view** and **3D spherical layout**.
- Navigate with minimap, run traversal presets, adjust layout/visual settings.
- Export graph + schema for reuse across sessions.

The `SPOTLIGHT_STEPS` configuration for the in‑app Launch tour encodes these four steps as explicit `title` and `body` strings so the spotlight card reads directly from this section (`canvas/src/features/spotlight/config.ts:1–52`).


### Detailed

The four phases below map directly to the in‑app Launch spotlight sequence defined in `SPOTLIGHT_STEPS` (`canvas/src/features/spotlight/config.ts:1–52`) and rendered by the top‑right spotlight card (`canvas/src/pages/Canvas.tsx:21–272`). The Panel Help Workflow section renders a matching overview using `WORKFLOW_STEP_COPY` / `getWorkflowStepBody` (`canvas/src/features/panels/config.ts`) and can be replayed from the rocket `Launch` toolbar button (`canvas/src/components/IconButton.tsx:22–43`) or Panel → Help → Launch.

1. Launch spotlight Step 1 – Parser scripts (offline normalization under `/parser-script`)
1.1. Author or run Python parser scripts in `/parser-script` (for example `parse-unicorn-investors-top-3-test-csv.py`, `parse-unicorn-investors-test-3d-viz.py`, `parse-ai-kg.py`) to transform raw CSV / JSON / JSON‑LD files from `/test-data` into normalized `GraphData` or `{nodes,edges}` JSON.
1.2. Use each script’s `--input` / `--output` / `--format` flags to import raw datasets and export graph JSON back into `/test-data` or `data/outputs` so the canvas app and regression tests can share the same parser outputs.

2. Launch spotlight Step 2 – Dataset import/export (CSV / JSON / JSON‑LD under `/test-data`)
2.1. Datasets such as `test-data/unicorn-investors-top-3-test.json` (raw `{nodes,edges}`) and `test-data/ai-kg-viz.json` (JSON‑LD) live under `/test-data` alongside their CSV sources (for example `test-data/unicorn-investors-top-3-test.csv`).
2.2. In the canvas app, the Toolbar `Load Data` button opens a file picker; when the user selects a CSV / JSON / JSON‑LD file from `/test-data`, the loader auto‑selects the appropriate parser spec (CSV, JSON‑LD, raw) and commits the resulting `GraphData` into the global store.
2.3. From the Panel view, users export the current graph as JSON, JSON‑LD, or combined CSV. Exporters enforce `test-data/` prefix and valid extensions, and auto‑suggest names from the active workflow preset via `getWorkflowPresetPipeline`.

3. Launch spotlight Step 3 – Schema‑config import/export (schema, presets, render settings under `/schema-config`)
3.1. Visual and behavioral presets for each dataset live under `/schema-config` (for example `schema-config/unicorn-investors-test-3d-viz-schema.json`, `schema-config/ai-kg-viz-schema.json`).

3.1.3. Within the Schema tab’s **Step 3.1. Apply presets from schema-config/** and nested **Step 3.1.1. 4. Launch spotlight Step 4 – Visualizing graphs for `(parser, dataset, schema)` combinations and exports
4.1. For each workflow, the Panel view tracks the triple `(parserSpecId, datasetFileName, schemaFileName)` so combinations can be re‑applied (for example: `jsonld` + `test-data/ai-kg-viz.json` + `schema-config/ai-kg-viz-schema.json`, or `raw`/`graph` + `test-data/unicorn-investors-top-3-test.json` + `schema-config/unicorn-investors-test-3d-viz-schema.json`).
4.2. Once parser and dataset succeed and a schema‑config is active, the canvas renders:
     - 2D D3 force‑directed view (`GraphCanvas`) for editing, inspection, and minimap‑driven navigation
     - 3D Three.js + React Three Fiber view (`ThreeGraph`) with Physics3D spherical layout when `canvasRenderMode` is `3d`, using `fibSphere` plus many‑body repulsion to assign `x,y,z` coordinates per tick
4.3. The user can toggle between 2D and 3D modes, run traversal presets (AI‑KG), tune layout and visual presets, and export both graph and schema so the same `(parser, dataset, schema)` combinations stay reproducible across sessions. Panel → Data provides JSON‑LD/JSON/combined CSV exports for GraphData, while Schema tabs in the Panel and bottom panel expose JSON/JSON‑LD/CSV schema export actions wired to `exportSchemaAsJSON`, `exportSchemaAsJsonLd`, and `exportSchemaAsCsv`.

5. Launch spotlight (config‑driven in‑app navigation under `/canvas`)
5.1. `SPOTLIGHT_STEPS` centralizes the four Launch spotlight steps as config: each step stores its `id` (1–4), `title`, `body`, `targetSelector`, `variant: 'primary' | 'secondary'`, and optional `requires` keys (`'parserOpen'`, `'renderOpen'`, `'traversalRan'`). The `title` and `body` for each spotlight step are authored explicitly to mirror the Interactive Product Tour & Onboarding Flow above (`Step 1 – Parse Data`, `Step 2 – Import / Export Datasets`, `Step 3 – Apply Schema Config`, `Step 4 – Visualize Graphs`), so the in‑app tour reads like the same four‑phase narrative. Panel Help still uses `WORKFLOW_STEP_COPY[id]` and `getWorkflowStepBody` to render a compact Workflow list that stays conceptually in sync with the spotlight card (`canvas/src/features/spotlight/config.ts:1–52`, `canvas/src/features/panels/config.ts`).
5.2. `LaunchSpotlight` maps requirement keys to store‑derived booleans in a small `requirementStatus` map and only enables the `Next` button when all configured `requires` are satisfied. `datasetLoaded` becomes true when graph data is present in the store, `schemaOpen` when the bottom panel Schema tab is active, `renderOpen` when the Render tab is active, and `traversalRan` when the AI‑KG traversal highlight has completed (or is treated as satisfied for non‑AI‑KG datasets). This replaces earlier inline checks such as `idx === 1 && seenLaunch` so gating logic is fully driven by config instead of hard‑coded indices (`canvas/src/pages/Canvas.tsx:21–110`).
5.3. On first load, the canvas shows a small launch spotlight card near the top‑right or anchored next to the current `targetSelector` via `useSpotlightAnchor`. It adapts its copy to the active context (Unicorn Investors vs AI‑KG), walks through connecting data, applying Unicorn/AI‑KG workflows, tuning render/layout (Unicorn 2D/3D presets or AI‑KG layers), and exploring/searching the graph. Users can drag the card, minimize it to a compact pill with `Dismiss` / `Reopen` actions, or step through `Next` / `Done`. Parser and render steps can automatically open the Panel and Render tabs so users land directly on the relevant controls, and the spotlight can be replayed from the Panel Help tab (`Launch` button), toggled on or off via the Panel Settings tab (`enableLaunchSpotlight`), or invoked directly from the canvas with Cmd/Ctrl+Shift+G (`canvas/src/features/spotlight/useSpotlightAnchor.ts`, `canvas/src/pages/Canvas.tsx:111–271`).

### AI Engineering Knowledge Graph Workflow (ai-kg-viz.json)
- Goal: render the AI Engineering Knowledge Graph JSON‑LD (`test-data/ai-kg-viz.json`) with styling aligned to the D3 prototype (`test-data/poc/ai-kg-viz.html`) and a minimal query traversal highlight driven by `graphRAGPath`.
- Steps:
  - 1) Start dev server: `cd canvas && pnpm install && pnpm run dev`; open `http://localhost:<port>/`.
  - 2) Open Panel → Help and scroll to the Workflow section.
  - 3) In the Data step, click the `AI KG Visualization` workflow button:
    - Sets the parser to the JSON‑LD spec via the branded `ParserId` (`parserId: toParserId('jsonld')`).
    - Records the workflow triple (`parserSpecId`, `datasetFileName`, `schemaFileName`) as branded `ParserId`/`DatasetPath`/`SchemaConfigPath` in `localStorage` under `kg:workflowPresets:lastApplied` so downstream tools can recover the exact combination using `getWorkflowPresetPipeline` (`canvas/src/features/panels/views/HelpView.tsx`, `canvas/src/features/panels/views/WorkflowSection.tsx`, `canvas/src/features/parsers/useParserWorkflowState.ts`).
  - 4) Click `Load Data` and select `/test-data/ai-kg-viz.json`:
    - Loader uses `bestMatch` to pick the JSON‑LD parser and calls `parseJsonLd` (`canvas/src/features/parsers/registry.ts:29–47`, `canvas/src/lib/graph/jsonld.ts:25–80`).
    - Updated `parseJsonLd` reads `@context` and treats properties with `@type: "@id"` (AI‑KG predicates such as `enables`, `requires`, `addresses`, `optimizes`, `extends`, `implements`, `uses`, `includes`, `generated_by`, `extends_to`, `alternative_to`, `adapts`, `enhances`, `collects`, `feeds`, `powered_by`, `evaluates`) as edge predicates when they point at other `air:*` node ids, generating `GraphEdge` entries for each relationship. The same predicates and edge‑generation rules are exercised end‑to‑end by the AI‑KG JSON‑LD and worker‑pipeline tests in `canvas/src/__tests__/roundtrip.test.ts` (`testJsonLdAiKgEdges`, `testJsonLdAiKgMemoryAgentsSlice`, `testJsonLdAiKgWorkerPipeline`, `testJsonLdAiKgTriplesMatchD3`), surfaced as `jsonld.aiKgEdges`, `jsonld.aiKgMemoryAgentsSlice`, `jsonld.aiKgWorkerPipeline`, and `jsonld.aiKgTriplesVsD3` in `canvas/src/tests/run.ts`. The memory/agents slice fixture also covers `graphRAGPath` traversal metadata to ensure it stays on nodes as properties rather than being misclassified as edges.
    - One or more nodes contain a `graphRAGPath` object with a `traverse` array; these ids define the traversal path used by the highlight behavior.
  - 5) Import the AI KG schema config from `/schema-config/ai-kg-viz-schema.json` via the Schema tab:
